# BaB — інструкція із запуску

Це перша частина програми: логін для співробітників + модуль **"Залишки"** (облік квітів/матеріалів на складі). Базу даних одразу створено під усі 4 модулі, тож наступні (букети, каса, постачальники) додамо без переробки бази.

Нижче — покроково, нічого програмувати не потрібно, тільки копіювати/вставляти.

## Крок 1. Створити базу даних (Supabase)

1. Зайдіть на **supabase.com** → Sign up (можна через Google).
2. New project → назвіть, наприклад, `flower-shop` → придумайте пароль до бази (збережіть його) → Create.
3. Зачекайте ~1-2 хв, поки проєкт створюється.
4. Зліва відкрийте **SQL Editor** → **New query**.
5. Відкрийте файл `supabase/schema.sql` з цього проєкту, скопіюйте весь вміст, вставте в редактор → натисніть **Run**.
6. Має з'явитись "Success. No rows returned" — це означає, що всі таблиці створено.

## Крок 2. Створити собі логін

1. Зліва → **Authentication** → **Users** → **Add user** → **Create new user**.
2. Вкажіть свій email і пароль (це буде логін у програму). Email confirm можна вимкнути в Settings → Auth, якщо не хочете підтверджувати пошту.

## Крок 3. Дізнатись ключі підключення

1. Зліва → **Project Settings** (шестерня) → **API**.
2. Скопіюйте **Project URL** та **anon public key** — вони знадобляться на кроці 5.

## Крок 4. Викласти код на GitHub

1. Зареєструйтесь на **github.com**, якщо ще немає акаунту.
2. New repository → назва, наприклад `flower-shop-app` → Create repository.
3. Натисніть **"uploading an existing file"** і перетягніть туди всі файли й папки з цього проєкту (збережіть структуру папок — `app`, `components`, `lib`, `supabase` мають лишитись папками).
4. Commit changes.

## Крок 5. Розгорнути на Vercel (сам сайт)

1. Зайдіть на **vercel.com** → Sign up через GitHub.
2. **Add New → Project** → оберіть свій репозиторій `flower-shop-app` → **Import**.
3. У розділі **Environment Variables** додайте два записи (значення з Кроку 3):
   - `NEXT_PUBLIC_SUPABASE_URL` = ваш Project URL
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY` = ваш anon public key
4. Натисніть **Deploy**. Через 1-2 хвилини отримаєте посилання виду `flower-shop-app.vercel.app` — це і є ваша програма, відкривається з будь-якого пристрою.

## Крок 6. Кожне наступне оновлення

Коли я даватиму нові файли (наступні модулі), потрібно буде лише:
1. Замінити/додати відповідні файли в GitHub-репозиторії.
2. Vercel сам перевипустить сайт за 1-2 хвилини — нічого додатково натискати не треба.

## Що вже працює

- Вхід за email/паролем
- Додавання/редагування/видалення матеріалів (квітів, стрічок тощо)
- Позначка "мало" коли залишок нижче встановленого порогу
- Огляд: кількість позицій, скільки товарів закінчується, загальна вартість складу

## Ролі: власник і продавець

У програмі два типи входу:

- **Власник** — бачить усе: залишки із закупівельними цінами, постачальників, собівартість і маржу букетів, огляд.
- **Продавець** — бачить лише "Продажі" (каса) і "Асортимент" (додати нову позицію / поповнити залишок кількості). Закупівельних цін не бачить ніде, і навіть не може їх встановити чи змінити — це заблоковано на рівні бази даних, не лише в інтерфейсі.

Коли ви створюєте нового користувача в Supabase (Authentication → Users → Add user), він **автоматично отримує роль "продавець"**. Щоб зробити когось власником (зазвичай — самого себе), виконайте в Supabase → SQL Editor:

```sql
update profiles set role = 'owner'
where id = (select id from auth.users where email = 'ваш-email@приклад.com');
```

(підставте свою пошту, якою логінитесь).

## Мережа магазинів

Тепер програма підтримує кілька точок:

1. Виконайте `migration_multi_store.sql` в Supabase (окремий файл, одноразово).
2. Зайдіть під власником → **"Магазини"** → додайте центральний склад і свої 6 точок.
3. Наявні раніше залишки автоматично потрапили у тимчасову локацію **"Тимчасовий склад (розподілити)"** — це зроблено, щоб нічого не загубити при міграції. Розподіліть їх по реальних магазинах вручну (поки що через "Залишки" в кожному магазині; повноцінне "Переміщення" — наступний крок).
4. У шапці зверху з'явився перемикач магazину — всі сторінки (Залишки, Продажі, Асортимент, Огляд) показують дані саме для обраної точки.
5. На сторінці "Магазини" також призначаєте, у яких точках може працювати кожен продавець (галочки в таблиці).

## Розпізнавання чеків постачальника (AI)

На сторінці "Поповнення складу" є кнопка "📷 Сканувати чек постачальника" — фотографуєте накладну, і Claude сам розпізнає товари, кількість і ціни, підставляючи їх у форму (ви завжди перевіряєте й підтверджуєте перед збереженням).

Щоб увімкнути:

1. Зайдіть на **console.anthropic.com** → зареєструйтесь → **Get API keys** → створіть ключ (почнеться на `sk-ant-...`) → скопіюйте його.
2. Поповніть баланс на невелику суму (сканування одного чека коштує частки копійки).
3. У Vercel → ваш проєкт → **Settings → Environment Variables** → додайте:
   - `ANTHROPIC_API_KEY` = ваш ключ з кроку 1 (без `NEXT_PUBLIC_` — цей ключ має лишатись у секреті на сервері)
4. **Redeploy** проєкт (Vercel → Deployments → три крапки на останньому → Redeploy).

Без цього кроку решта програми працює як і раніше — просто кнопка сканування покаже помилку.
